Summary for the 70th meeting of Ecma TC39

Ecma Technical Committee 39 held a three day meeting hosted by Stripe in Berlin, Germany on June 4th to June 6th, 2019. Agenda can be found here.

  • June 4, 2019 Meeting Notes
  • June 5, 2019 Meeting Notes
  • June 6, 2019 Meeting Notes

New Proposals

TC39 advanced 3 new proposals to Stage 1, and one proposal went straight to Stage 2.

Advanced to Stage 1

  • Emitter
  • evalable
  • Host compile value adjustment (combined with evalable)

Advanced from Stage 0 to 2

  • Array.isTemplateObject

Existing Proposals

TC39 discussed 11 proposals for stage advancement and had updates presentations on 4 proposals

Advanced to Stage 2 from Stage 1

  • For-in enumeration order
  • ECMA-402: Intl.DisplayNames
  • Optional Chaining for Stage 2
  • Nullish Coalescing for Stage 2

Advanced to Stage 3 from Stage 2

  • Top-level await
  • WeakRefs

Advanced to Stage 4 from Stage 3

  • Dynamic import()
  • BigInt

Needs more discussion/information

Following proposals did not reach consensus for stage advancement.

  • Annex B reform
  • JavaScript Standard Library for Stage 2
    • JavaScript Standard Library for Stage 2 [ part 2? ]
  • A JavaScript Commons

Updates

Updates on proposals that are not seeking stage advancement

  • Atomics.waitAsync asking for stage 3 reviewers
  • How should we specify Jobs precisely?
    • Continuation of Layering Job Queue
  • Trusted types
  • Promise.any

Changes to existing specification

TC39 discussed 7 Pull Requests to the existing specification

Consensus Reached Pull Requests

  • Let all early errors be Syntax Errors
  • eval(nonString) should not side effect
  • Set "name" property for anonymous functions
  • ECMA-402: Add quarter option
  • ECMA-402: add dayPeriod option
  • ECMA-402: add millisecondsDigits option
  • Lookup constructor.resolve only once in PerformPromise{All, Race}

Needs more discussion/information

  • none
